# Краткое руководство по развёртыванию сервера, CMS и VPN

## Что делает этот проект

Набор скриптов автоматически устанавливает и настраивает на чистом сервере Ubuntu 24.04:

- **Веб-сервер** (Nginx, PHP 8.3, MariaDB) с поддержкой SSL (Let's Encrypt или готовые сертификаты).
- **Собственную CMS** с админ-панелью (управление страницами, пользователями, загрузка файлов, статистика посещений, аналитика).
- **AI-генератор контента** на базе DeepSeek (кнопка в редакторе, история запросов).
- **VPN-сервер** (3X-UI, VLESS+TLS) на том же домене и порту 443, не мешая работе сайта.

Все скрипты идемпотентны: повторный запуск не ломает уже настроенную систему.

---

## Необходимые файлы

Скопируйте в одну директорию на сервере:

- `lib.sh` – общая библиотека (обязательна для всех).
- `init.sh` – базовая подготовка сервера.
- `cms.sh` – установка CMS, админки, AI.
- `vpn.sh` – настройка VPN (опционально). ⚠️ Важное предупреждение. В данный момент не доделан и не выполняет все возложенные на него функции.

## Последовательность действий

### 1. Базовая подготовка сервера

```bash
chmod +x init.sh
./init.sh
```

В интерактивном меню выберите **цель 1 (Сайт)** или **4 (сайт + VPN)**, если планируете ставить VPN позже. Скрипт установит Nginx, PHP, MariaDB, настроит базовую безопасность (UFW, fail2ban) и cron.

> После выполнения `init.sh` сервер готов к установке CMS.

### 2. Установка CMS, админ-панели и AI

```bash
chmod +x cms.sh
./cms.sh
```

Скрипт запросит:
- Доменное имя (например, `example.com`).
- Email для сертификатов и уведомлений.
- Тип SSL: `letsencrypt` (автоматическое получение) или `existing` (путь к готовым сертификатам).

После ответов на вопросы скрипт:
- Создаст базу данных и пользователя.
- Настроит виртуальный хост Nginx с SSL.
- Установит CMS, админ-панель, трекер посещений, TinyMCE, AI DeepSeek.

**Итог:**
- Сайт доступен по `https://ваш_домен`
- Админ-панель: `https://ваш_домен/admin`
- Логин: `admin`, пароль выводится в конце (сохраните его).

### 3. (Опционально) Размещение готовых файлов сайта

Если у вас уже есть HTML/PHP-файлы сайта:
- Скопируйте их в `/var/www/ваш_домен/`.
- Зайдите в админ-панель → раздел «Аналитика».
- Выберите нужные файлы и нажмите «Встроить аналитику» – система сама добавит код сбора статистики.

### 4. (Опционально) Установка VPN

```bash
chmod +x vpn.sh
./vpn.sh
```

Скрипт:
- Настроит Nginx на локальный порт (10443) для fallback.
- Установит 3X-UI, создаст inbound VLESS+TLS на порту 443.
- Сгенерирует случайные параметры панели (порт, путь, логин, пароль).
- Откроет порты в UFW.

**Итог:**
- Панель управления 3X-UI: `https://ваш_домен:ПОРТ/ПУТЬ` (логин/пароль выводятся).
- Параметры VPN-клиента (VLESS, порт 443, UUID, flow, SNI, ALPN) – в консоли.

## Важные замечания

- **Все скрипты должны запускаться от root** (или через sudo).
- **Не прерывайте выполнение** скриптов – они могут оставить систему в нестабильном состоянии.
- **Сохраняйте вывод** – в конце каждого скрипта выводятся пароли и ссылки.
- **Логи** пишутся в `/var/log/setup.log`.
- При повторном запуске используйте флаг `--force`, чтобы перезаписать файлы (но не данные в БД).

## Требования к серверу

- **ОС:** Ubuntu 24.04 LTS (чистая установка).
- **Домен:** должен быть делегирован на IP сервера (для Let's Encrypt).
- **Порты:** 22 (SSH), 80, 443 должны быть доступны извне.
- **Минимум:** 1 GB RAM, 1 vCPU, 10 GB диска.

## Устранение типичных проблем

| Проблема | Решение |
|----------|---------|
| Сайт не открывается после `cms.sh` | Проверьте, что Nginx запущен: `systemctl status nginx`. Посмотрите логи: `tail -30 /var/log/nginx/ваш_домен_error.log`. |
| Админ-панель не открывается (500 ошибка) | Проверьте права на файлы: `chown -R www-data:www-data /var/www/ваш_домен`. Проверьте логи PHP-FPM: `tail -30 /var/log/php8.3-fpm.log`. |
| Не работают логин/пароль в 3X-UI | Выполните сброс: `x-ui setting -username admin -password admin123` и перезапустите `systemctl restart x-ui`. |
| VPN не подключается | Убедитесь, что порт 443 открыт и Xray слушает: `ss -tlnp | grep 443`. Проверьте логи Xray: `journalctl -u x-ui -n 30`. |
